# Contributing to SafeLine

Thank you for your interest in contributing to SafeLine! Whether you are reporting bugs, suggesting features, or working on code, your contributions are welcome.

## Contribution Guidelines
- **Be respectful**: Treat everyone with kindness and respect, and maintain a collaborative atmosphere.
- **Be clear and concise**: When reporting issues or submitting pull requests, provide detailed information on the problem or feature.

## Code Standards
- Follow [Google Java Style Guide](https://google.github.io/styleguide/jsguide.html) for Java code.
- Ensure that your code is well-documented with comments and adheres to existing code style within the project.

## Testing Requirements
- Write unit tests for any new features or bug fixes.
- Ensure that all tests pass before submitting your pull request.
- Run the existing test suite to ensure that your changes do not introduce new issues.

## Pull Request Process
1. **Fork the repository**: Create your own copy of the repository by forking it.
2. **Create a new branch**: Use a descriptive name for your branch related to the feature or fix you are working on.
3. **Commit your changes**: Include a clear commit message describing your changes.
4. **Push your changes**: Push your branch to your forked repository.
5. **Create a pull request**: Submit a pull request to merge your changes into the main repository. Ensure to describe your changes and link any related issues.
6. **Review process**: Once your pull request is submitted, it will be reviewed by the project maintainers. Be open to feedback and necessary changes before merging.

Thank you for contributing to SafeLine! We appreciate your efforts to improve our project.
